Protection modules

Simple controls. Strong privacy layers.

Zenith combines network blocking, cosmetic filtering, privacy helpers, and per-site controls in one Chrome MV3 extension.

```
🚫

Ad blocking

Blocks compatible ad requests with Chrome declarativeNetRequest and hides ad containers using cosmetic filtering.

🔍

Tracker protection

Blocks known trackers and includes smart tracker learning for repeated third-party tracking behavior.

🔒

Fingerprint protection

Reduces selected browser fingerprinting signals such as canvas, WebGL, audio, navigator data, and privacy signals.

🍪

Cookie popup handling

Rejects or hides supported cookie consent banners and common consent management popups.

🧹

Annoyance blocking

Hides newsletter prompts, app banners, chat widgets, overlays, floating videos, and similar interruptions.

📡

Network Logger

Shows blocked requests in a clean live logger with filtering, request type labels, and local-only logs.

```

Install

Load Zenith locally in Chrome.

Zenith can be tested as an unpacked Chrome extension during development.

```
1

Download or clone the project

Download the ZIP from GitHub or clone the repository using git clone.

2

Open Chrome extensions

Go to chrome://extensions/ and enable Developer mode.

3

Load unpacked

Click Load unpacked and select the Zenith project folder.

4

Test the extension

Open the popup, dashboard, and Network Logger. Confirm there are no Chrome extension errors.

```
Privacy first

No remote analytics. No acceptable ads program.

Zenith stores settings and local logs on the user’s device. Filter lists are treated as text data, not executable code.

```
🔐

Local settings

Whitelist domains, per-site controls, counters, and logs are stored locally using Chrome storage.

🧱

CSP-safe UI

Extension pages avoid remote fonts, remote scripts, eval, and unnecessary external dependencies.

🚫

No acceptable ads

Zenith does not include a paid acceptable ads program or advertising whitelist.
